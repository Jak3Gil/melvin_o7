## Automatic Build & Deploy

The website will be automatically built and deployed when you push to GitHub. Follow these steps:

### Step 1: Push to GitHub
```bash
git add .
git commit -m "Add web interface for GitHub Pages"
git push origin main
```

### Step 2: Enable GitHub Pages
1. Go to your repository on GitHub: `https://github.com/[username]/[repository-name]`
2. Click **Settings** (top menu)
3. Scroll to **Pages** (left sidebar)
4. Under "Source", select **"GitHub Actions"**
5. Click **Save**

### Step 3: Wait for Build
1. Go to the **Actions** tab in your repository
2. You'll see the "Deploy to GitHub Pages" workflow running
3. It will:
   - Install Emscripten
   - Compile `melvin.c` and `melvin_wasm.c` to WebAssembly
   - Deploy to GitHub Pages
4. Wait 2-3 minutes for completion

### Step 4: Access Your Site
Your site will be live at:
```
https://[username].github.io/[repository-name]/
```

For example: `https://yourusername.github.io/melvin_o7/`

## Manual Build (Optional)

If you want to build locally first (requires Emscripten):

### Install Emscripten
1. Download: https://emscripten.org/docs/getting_started/downloads.html
2. Follow installation instructions for your OS

### Build
**Windows:**
```bash
build_wasm.bat
```

**Linux/Mac:**
```bash
chmod +x build_wasm.sh
./build_wasm.sh
```

This creates `web/melvin.js` and `web/melvin.wasm` locally.

## Troubleshooting

- **Build fails in GitHub Actions**: Check the Actions tab for error messages
- **404 on site**: Make sure GitHub Pages source is set to "GitHub Actions"
- **WASM not loading**: The build should create both `melvin.js` and `melvin.wasm` in `web/`
