# Deploy Melvin Chat to GitHub Pages

## Quick Deploy (Automatic)

1. **Push to GitHub:**
   ```bash
   git add .
   git commit -m "Add web interface for GitHub Pages"
   git push origin main
   ```

2. **Enable GitHub Pages:**
   - Go to your repository on GitHub
   - Click **Settings** â†’ **Pages**
   - Under "Source", select **"GitHub Actions"**
   - Save

3. **Wait for deployment:**
   - Go to **Actions** tab
   - The workflow will automatically build WebAssembly and deploy
   - Your site will be live at: `https://[username].github.io/[repository-name]/`

## Manual Build (If Needed)

If you want to test locally first:

### Windows:
```bash
build_wasm.bat
```

### Linux/Mac:
```bash
chmod +x build_wasm.sh
./build_wasm.sh
```

This creates `web/melvin.js` and `web/melvin.wasm` files.

## File Structure

The `web/` directory contains everything needed:
- `index.html` - Main page
- `style.css` - Styling  
- `app.js` - Frontend JavaScript
- `melvin.js` - WebAssembly glue (generated by build)
- `melvin.wasm` - WebAssembly binary (generated by build)

## Testing Locally

After building, test with a local server:

```bash
cd web
python -m http.server 8000
# Or: npx http-server -p 8000
```

Then open `http://localhost:8000` in your browser.

## Troubleshooting

- **Build fails**: Make sure Emscripten is installed (GitHub Actions handles this automatically)
- **404 errors**: Check that GitHub Pages is set to use "GitHub Actions" as source
- **WASM not loading**: Ensure `melvin.js` and `melvin.wasm` are in the `web/` directory
